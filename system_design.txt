你是一个20年工作经验的选品专家, 给你一个案例:
数据源:
1. 从 TikTok Shop 网站获取商品详情页数据（示例链接：https://shop.tiktok.com/view/product/1730600050142774738?region=PH&locale=zh-CN&source=agency）
2. 可采集的信息包括：商品图片、标题、价格、销量、评分、库存状态、店铺信息等。
3. 将采集到的数据同步到 global_picker​ 系统。
功能扩展​:
1. 基于 TikTok 商品图片，自动在 拼多多​ 平台检索相似或相同商品，并展示对应信息（包括价格、销量、店铺名、图片等）。
2. 支持人工比对与标记，方便判断是否存在套利空间。
3. 根据tiktok提供的商品价格和拼多多售价并计算出该产品利润
系统目标​:
1. 为选品团队提供直观的比价与选品工具，帮助快速发现跨平台利润机会。
2. 支持批量导入 TikTok 商品链接，自动化抓取并展示数据。
3. 提供筛选、排序、收藏、导出等功能。
前端界面设计要求​:
1. 界面布局清晰，分为 TikTok 商品信息区、拼多多比价结果区、操作控制区。
2. 支持响应式设计，可在 PC 端及平板端流畅使用。
3. 提供商品图片预览、价格对比表、销量趋势图（可选）、利润率计算模块。
4. 操作区包含：添加商品链接、批量导入、刷新数据、导出报表、标记状态（已选/待定/放弃）。
5. 数据加载状态与错误提示需明确显示。
6. 支持多语言切换（至少中英文）。

技术要求
1. 前后端分离, 后端独立部署, python语言，提供数据接口，包括：商品抓取、比价查询、状态管理、报表导出等。
2. 数据接口需支持 分页、过滤、排序​ 参数，减少单次请求数据量，提高性能。
3. 图片资源使用 CDN 加速，前端实现懒加载与点击放大预览。
4. 数据库建议采用 MySQL + Redis 缓存，提高数据读取速度。
5. 系统需具备日志监控与异常报警能力，方便运维排查。
