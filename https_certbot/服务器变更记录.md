## 变更概述

将服务器从 HTTP 升级为 HTTPS，使用 Let's Encrypt 免费 SSL 证书，通过 certbot 工具自动配置和续期。

## 变更清单

### 1. 安装 Certbot

**操作**: 安装 Let's Encrypt 证书管理工具

```bash
# Ubuntu/Debian
apt update
apt install certbot python3-certbot-nginx

# 或 CentOS/RHEL
yum install epel-release
yum install certbot python3-certbot-nginx
```

**影响**: 新增系统包，无服务中断

---

### 2. 获取 SSL 证书

**操作**: 申请 Let's Encrypt 证书

```bash
# 方式 1: standalone 模式（需要临时停止 nginx）
certbot certonly --standalone -d www.globalpicker.com -d globalpicker.com \
  --non-interactive --agree-tos -m <邮箱>

# 方式 2: webroot 模式（无需停止 nginx）
certbot certonly --webroot -w /var/www/html \
  -d www.globalpicker.com -d globalpicker.com
```

**证书文件位置**:
- 证书: `/etc/letsencrypt/live/www.globalpicker.com/fullchain.pem`
- 私钥: `/etc/letsencrypt/live/www.globalpicker.com/privkey.pem`
- 配置: `/etc/letsencrypt/renewal/www.globalpicker.com.conf`

**影响**: 需要 80 端口可访问（standalone 模式需临时停止 nginx）

---

### 3. Nginx 配置变更

**操作**: 修改 Nginx 配置，启用 HTTPS 并强制 HTTP 跳转

**变更前**: 仅监听 80 端口的 HTTP 配置

**变更后**:
```nginx
# HTTP 服务器 - 重定向到 HTTPS
server {
    listen 80;
    server_name www.globalpicker.com globalpicker.com;
    
    location /.well-known/acme-challenge/ {
        root /var/www/html;
    }
    
    location / {
        return 301 https://$host$request_uri;
    }
}

# HTTPS 服务器
server {
    listen 443 ssl http2;
    server_name www.globalpicker.com globalpicker.com;
    
    ssl_certificate /etc/letsencrypt/live/www.globalpicker.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/www.globalpicker.com/privkey.pem;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    ssl_stapling on;
    ssl_stapling_verify on;
    
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Frame-Options SAMEORIGIN always;
    add_header X-Content-Type-Options nosniff always;
    
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**影响**: 
- 所有 HTTP 请求自动 301 跳转到 HTTPS
- Nginx 需要重启（短暂中断，约 1-2 秒）
- 后端服务端口需为 8000（根据实际情况调整）

**备份文件**:
- `/etc/nginx/nginx.conf.pre-https-backup`
- `/etc/nginx/nginx.conf.backup-YYYYMMDD`

---

### 4. 设置自动续期

**操作**: 配置 certbot 自动续期，防止证书过期

```bash
# 测试续期
certbot renew --dry-run

# 添加定时任务（每天凌晨 2 点检查）
0 2 * * * /usr/bin/certbot renew --quiet --post-hook "systemctl reload nginx"
```

**影响**: 新增 crontab 任务，证书过期前 30 天自动续期

---

### 5. 阿里云安全组配置

**操作**: 确保 443 端口已开放

**需要开放的端口**:
- 80 (HTTP) - 用于证书验证和 HTTP 跳转
- 443 (HTTPS) - 用于 HTTPS 服务

**影响**: 如未开放 443 端口，HTTPS 将无法访问

## 注意事项

1. **证书有效期**: Let's Encrypt 证书有效期 90 天，务必确保自动续期正常
2. **端口要求**: 80 和 443 端口必须开放
3. **域名解析**: www.globalpicker.com 和 globalpicker.com 都需解析到 47.238.72.198
4. **后端端口**: Nginx 配置中 proxy_pass 指向 127.0.0.1:8000，需与实际后端端口一致
5. **监控**: 建议设置证书过期告警（certbot 会发邮件通知）
