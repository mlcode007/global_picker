## 1. TikTok 采集 · 越南等价格解析（避免 0.00）

**问题**：`sale_price_format` 在 VND 下为千分位点号（如 `9.319.000`），`Decimal` 无法解析多个小数点，导致标价为 0。

**改动**：

- `backend/app/workers/tiktok_crawler.py`
  - `_safe_decimal`：纯数字串；多「.」按千分位处理；保留常见 `1,234.56` / `1.234,56` 等。
  - `_parse_remix_component_data`：优先 `sale_price_decimal` / `single_product_price_decimal`，再 `sale_price_format`；补充 `originalPrice`（`origin_price_decimal` / `origin_price_format`）。
  - `_tiktok_price_changed` + 采集成功后若标价变化则 `refresh_product_profit_from_primary_pdd`（存在主参照拼多多时重算预估利润）。

**关联**：`backend/app/services/pdd_service.py` 新增 `refresh_product_profit_from_primary_pdd`。

## 4. 列表页 · 自动拍照购（`ProductList.vue`）

### 4.1 云手机检查

- 遍历账号下**非离线**云手机，逐台 `checkHealth`；最多 **3 轮**；不再使用大块多行日志，仅更新单行 `stepText`。

### 4.2 失败重试

- 单商品最多 **3 次**尝试；第 2、3 次清空 `photoHealthPhoneCache` 强制重做设备检查；有任务 ID 时走 `retryTask`，否则再次 `createTask`。

### 4.3 「每笔最多入库」语义（重要）

- **本批**：对勾选且**有主图**的 TikTok 商品**全部依次**执行拍照购。
- **数字 N**：仅表示接口 **`max_candidates`**（每笔最多入库 **N 条拼多多**），**不再**用 N 截断本批 TikTok 行数。
- 工具栏文案：**每笔最多入库 [N] 条拼多多**；`resolvePhotoMaxCandidates()` 仅用于 `max_candidates`。
