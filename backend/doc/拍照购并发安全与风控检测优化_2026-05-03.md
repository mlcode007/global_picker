## 二、Redis 分布式设备锁

### 步骤10：新增 device_lock.py 模块

**文件**：`backend/app/workers/pdd_photo/device_lock.py`（新增）

**功能**：
- Redis SETNX + TTL 分布式锁
- TTL 120s 自动过期（任务卡死时自动释放）
- 心跳续期机制（每 30s 续期）
- Lua 脚本安全释放（仅锁持有者可释放）
- 僵尸锁检测与强制释放

**核心方法**：
| 方法 | 说明 |
|------|------|
| `acquire(timeout)` | 尝试获取锁，超时返回 False |
| `release()` | 释放锁（Lua 脚本保证安全） |
| `heartbeat()` | 心跳续期，延长 TTL |
| `get_zombie_locks()` | 获取超过指定时间无心跳的锁 |
| `force_release()` | 强制释放锁（管理员操作） |

### worker.py 集成 Redis 锁

**文件**：`backend/app/workers/pdd_photo/worker.py`

**修改**：
1. 设备获取后尝试 Redis 锁（30s 超时）
2. Redis 锁失败时立即释放 DB 设备状态
3. 启动心跳续期线程（每 30s 续期）
4. finally 块中停止心跳 → 释放 Redis 锁 → 释放 DB 设备
5. 添加 `_redis_lock_acquired` 标记避免重复释放
6. 心跳线程添加 try/except，失败时记录日志

### device_manager.py 超时优化

**文件**：`backend/app/workers/pdd_photo/device_manager.py`

**修改**：
- `ACQUIRE_WAIT_SECONDS`：600s → 30s

**效果**：用户卡在"分配设备"的时间从 10 分钟降到 30 秒

## 三、并发安全设计

### 保护机制

| 场景 | 保护机制 | 状态 |
|------|---------|------|
| 同一用户多任务并发 | DB `FOR UPDATE SKIP LOCKED` + Redis 分布式锁 | ✅ 安全 |
| 不同用户设备隔离 | `user_id` 过滤 `CloudPhonePool.created_by` | ✅ 安全 |
| 任务卡死设备不释放 | Redis TTL 120s 自动过期 | ✅ 安全 |
| 进程被 kill | Redis 锁不依赖 finally，TTL 自动回收 | ✅ 安全 |
| 心跳续期 | 每 30s 续期，正常任务不会过期 | ✅ 安全 |
| 快速失败 | 超时从 600s 降到 30s | ✅ 安全 |
| 用户多设备 | 每个设备独立 serial + 独立 Redis 锁 | ✅ 安全 |

### 工作流程

```
try:
    1. 获取 DB 设备 (acquire_device)
    2. 获取 Redis 锁 (device_lock.acquire, 30s 超时)
       └─ 失败 → 释放 DB 设备 → return
       └─ 成功 → _redis_lock_acquired = True
    3. 启动心跳线程（每 30s 续期）
    4. 执行拍照购流程
finally:
    停止心跳 → 释放 Redis 锁 → 释放 DB 设备
    (仅当 _redis_lock_acquired=True 时才执行)
```
